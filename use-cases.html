Beginner

1. Simple Research Assistant

Purpose: Create an AI agent that researches topics and provides structured summaries.

Blocks Needed:

Agent Response
Workflow Steps:
  1. Add Agent block
  2. Select model: gemma2:2b or llama3.2:3b
  3. Configure system prompt (see below)
  4. Add Response block
  5. Test in Chat panel
System Prompt: You are a research assistant. When given a topic, provide: 1. Brief overview (2-3 sentences) 2. Key points (3-5 bullet points) 3. Potential applications 4. Related topics to explore Keep responses concise and well-structured.
Try asking: "Research quantum computing for business applications"

2. Email Response Generator

Purpose: Draft professional email responses based on incoming message context.

Blocks Needed:

Agent Response
Workflow Steps:
  1. Create new workflow
  2. Add Agent block
  3. Set system prompt for email tone
  4. Test with sample emails
System Prompt: You are a professional email assistant. Given an incoming email, draft a clear, professional response that: - Acknowledges the sender's message - Addresses their questions or concerns - Maintains appropriate business tone - Keeps it concise (under 200 words) - Ends with appropriate closing Format the response ready to send.
Input example: Paste an email asking about meeting availability, and get a draft response.

Intermediate

3. Data Analyzer with Structured Output

Purpose: Analyze data and return results in consistent JSON format.

Blocks Needed:

Agent Response
Workflow Steps:
  1. Add Agent block
  2. Configure system prompt
  3. Click "Add Response Format"
  4. Use AI to generate schema: "create schema with fields: summary, key_findings, recommendations, confidence_score"
  5. Test with sample data
System Prompt: You are a data analysis assistant. Analyze the provided information and extract: - Summary: Brief overview - Key findings: Important insights (list) - Recommendations: Actionable suggestions - Confidence score: 1-10 rating of data quality Be objective and data-driven.
Try with: Paste sales data, survey results, or meeting notes to get structured analysis.

4. Content Repurposer

Purpose: Transform content between formats (blog to social, long to short, etc.).

Blocks Needed:

Agent Router Response
Workflow Steps:
  1. Add Agent block for content analysis
  2. Add Router block to determine format
  3. Add multiple Agent blocks for each format
  4. Connect outputs to Response
System Prompt: You are a content repurposing specialist. Transform the provided content into the requested format: - Twitter thread: 5-7 tweets with hooks - LinkedIn post: Professional, 150-200 words - Email newsletter: Engaging, action-oriented - Blog summary: 3 paragraph overview Maintain key messages while adapting tone and length.
Use for: Turn blog posts into social media content automatically.

5. Meeting Notes Processor

Purpose: Convert raw meeting transcripts into action items and summaries.

Blocks Needed:

Agent Response
Workflow Steps:
  1. Create Agent with specialized prompt
  2. Add structured output schema
  3. Schema fields: decisions, action_items, next_steps, attendees
  4. Test with meeting transcript
System Prompt: Extract structured information from meeting notes: DECISIONS: Key decisions made ACTION ITEMS: Who needs to do what by when NEXT STEPS: Follow-up actions required KEY DISCUSSIONS: Main topics covered ATTENDEES: Who was present Format clearly with owners and deadlines where mentioned.
Saves time: Turn 30-minute meeting recordings into 2-minute summaries.

6. Multi-Agent Research System

Purpose: Multiple specialized agents collaborate on complex research tasks.

Blocks Needed:

Agent (x3) Router Condition Response
Workflow Steps:
  1. Agent 1: Initial research & fact-finding

Complexity level: This requires understanding of workflow logic and agent coordination. Start with simpler use cases first.

Best Practices

System Prompt Tips

  • Be specific: Define exact output format and requirements
  • Include examples: Show the AI what good output looks like
  • Set constraints: Word limits, tone, structure
  • Define role: "You are a..." helps set context

Workflow Design Tips

  • Start simple: Single agent workflows first
  • Test incrementally: Add blocks one at a time
  • Use structured output: JSON schemas for consistent results
  • Name blocks clearly: Makes debugging easier
  • Document your prompts: Save working prompts for reuse

Model Selection Guide

gemma2:2b - Best for:
  • Simple text generation
  • Quick responses needed
  • High-volume processing
  • Low resource usage required
llama3.2:3b - Best for:
  • Balanced performance
  • General purpose workflows
  • Moderate complexity tasks
  • Good quality/speed tradeoff
llama3.1:8b - Best for:
  • Complex reasoning
  • Technical content
  • High accuracy requirements
  • Multi-step analysis
